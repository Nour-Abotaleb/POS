Description

The MultiPOS Module lets restaurants register and manage multiple POS devices from one centralized dashboard. Track sales and performance for each device, approve or reject new device registrations, and access detailed machine-wise reports. Ideal for restaurants using multiple terminals across branches or service areas.

The MultiPOS module enables TableTrack to support multiple Point of Sale (POS) terminals simultaneously, allowing restaurants to manage orders from different locations, devices, or stations. This module is perfect for restaurants with multiple service points, food courts, and establishments looking to scale their operations efficiently.

Staff can process orders, manage tables, and handle payments from multiple POS terminals. All orders are synchronized in real-time across all devices, ensuring seamless coordination between different service points and streamlining the entire restaurant operations.

Key Features

  • Multiple POS terminal support
  • Real-time synchronization across devices
  • Centralized order management
  • Device registration with approval workflow
  • Machine-wise sales and performance tracking
  • Detailed machine-wise reports
  • Branch-wise machine management
  • Package-based device limits

Integration Guide

Update Order Model

Add relationship to app/Models/Order.php:

use Modules\MultiPOS\Entities\PosMachine;

public function posMachine()
{
    return $this->belongsTo(PosMachine::class, 'pos_machine_id');
}

Add Middleware to POS Routes

In your route files, add the middleware:

Route::middleware(['auth', 'verified', 'pos.machine'])->group(function () {
    Route::get('/pos', [PosController::class, 'index'])->name('pos.index');
    // ... other POS routes
});

Update POS Livewire Component

In your POS component, update the order creation:

// After creating order
$order->pos_machine_id = pos_machine_id();
$order->save();

Helper Functions

  • pos_machine() - Get current PosMachine instance
  • pos_machine_id() - Get current machine ID
  • is_pos_registered() - Check if device is registered
  • pos_machine_alias() - Get machine alias

API Endpoints

  • GET /pos/claim - Show registration form
  • POST /pos/claim - Register device
  • GET /pos/claim/check - Check registration status
  • GET /admin/multi-pos/machines - List machines
  • POST /admin/multi-pos/machines/{id}/approve - Approve machine
  • POST /admin/multi-pos/machines/{id}/decline - Decline machine

Usage Instructions

For Users

  1. Log into POS page
  2. If device not registered, you'll see registration prompt
  3. Click "Register Device" → Select branch → Enter alias
  4. Wait for admin approval (or auto-approved based on config)
  5. Device is now tracked for all orders

For Admins

  1. Go to MultiPOS → Machines or Settings
  2. View pending machines in "Pending Approvals"
  3. Approve or decline registrations
  4. Manage active machines (edit alias, delete, view stats)
  5. Monitor machine activity and last-seen status
  6. Access detailed machine-wise reports

For Developers

  • • Use pos_machine() helper to get current machine
  • • Use middleware pos.machine on POS routes
  • • Track orders with pos_machine_id
  • • Use is_pos_registered() to check status
  • • Access machine statistics and reports via API
