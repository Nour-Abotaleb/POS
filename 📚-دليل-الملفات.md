# 📚 دليل الملفات - حل مشاكل النشر

## 📋 نظرة عامة

هذا المشروع يحتوي على مجموعة شاملة من الأدوات والأدلة لحل مشاكل النشر على السيرفر، خاصة مشكلة MultiPOS.

---

## 🚀 ملفات التنفيذ (Scripts)

### 1. `quick-fix.sh` ⚡
**الغرض:** حل سريع شامل لمعظم المشاكل

**متى تستخدمه:**
- بعد رفع الملفات على السيرفر
- عند ظهور أي خطأ في الـ cache
- بعد تحديث الكود

**الاستخدام:**
```bash
chmod +x quick-fix.sh
./quick-fix.sh
```

**ماذا يفعل:**
- ينظف جميع الـ caches
- يحذف ملفات bootstrap cache
- يعيد بناء autoload
- يكتشف الـ modules
- يصلح الصلاحيات
- يحسن التطبيق

---

### 2. `fix-multipos-issue.sh` 🔧
**الغرض:** حل متخصص لمشكلة MultiPOS

**متى تستخدمه:**
- عند ظهور خطأ MultiPOSServiceProvider
- عند ظهور خطأ translator

**الاستخدام:**
```bash
chmod +x fix-multipos-issue.sh
./fix-multipos-issue.sh
```

**ماذا يفعل:**
- ينظف الـ caches
- يحذف ملفات modules cache
- يعيد اكتشاف الـ modules
- يحسن التطبيق

---

### 3. `deploy.sh` 🚀
**الغرض:** نشر كامل للتطبيق

**متى تستخدمه:**
- عند رفع تحديثات جديدة
- للنشر الآلي من Git

**الاستخدام:**
```bash
chmod +x deploy.sh
./deploy.sh
```

**ماذا يفعل:**
- يعمل backup
- يفعل maintenance mode
- يسحب آخر تحديثات من Git
- يثبت dependencies
- يشغل migrations
- ينظف الـ caches
- يبني الـ assets
- يصلح الصلاحيات
- يلغي maintenance mode

---

### 4. `backup.sh` 💾
**الغرض:** عمل نسخة احتياطية

**متى تستخدمه:**
- قبل أي تحديث مهم
- بشكل دوري (يومي/أسبوعي)

**الاستخدام:**
```bash
chmod +x backup.sh
./backup.sh
```

**ماذا يفعل:**
- ينسخ قاعدة البيانات
- ينسخ ملفات storage
- يحتفظ بآخر 7 نسخ
